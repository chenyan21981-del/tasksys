# TaskSys 系统使用指南（最新版）

## 1. 系统角色与登录

- 超级管理员（superadmin）：可管理账号、重置密码、分配权限
- 管理员（admin）：可创建/管理任务、查看看板
- 普通账号：按权限查看和处理分配任务

登录方式：
1. 打开系统首页（如 `http://127.0.0.1:8090/`）
2. 输入用户名和密码
3. 首次管理员登录后建议立即改密

---

## 2. 主任务流程（Web）

### 2.1 创建主任务
在「进行中任务」或「新建任务」中填写：
- 名称（必填）
- 文档链接（选填）
- 提出人（建议 `@username`）
- 类型、优先级、项目
- 开始时间/结束时间
- 抄送与备注（选填）

提交后系统会生成任务编号（如 `18G-1001`）。

### 2.2 流转与完成
- 可将任务流转给下一责任人
- 若无下一责任人，可直接完成任务
- 任务过程可持续追加备注，形成留痕

### 2.3 查询与导出
- 按创建人、提出人、关键词、责任人筛选
- 支持导出模板、批量导入、导出全部任务

---

## 3. 子任务与提醒

- 子任务支持责任人、状态、到期时间管理
- 机器人支持每日提醒和到期前提醒（按配置）
- 建议在任务拆分时明确：负责人 + 截止时间 + 验收标准

---

## 4. 看板与数据分析

系统提供两类分析：

1. **任务效率分析面板**：按日/周/月/全量查看项目数据
2. **单人数据面板**：查看当前登录账号的工作数据

建议每周固定复盘：
- 逾期率
- 完成率
- 平均流转时长
- 高优先级任务堆积情况

---

## 5. Telegram 指令（Bot）

常用指令示例：

- 新建任务：
  `/task 创建主任务 名称=登录优化 类型=优化 提出人=@pm 项目=18game 开始时间=2026-04-15 10:00 结束时间=2026-04-16 18:00`
- 流转任务：
  `/task 流转任务 #18G-1001 给 @alice`
- 完成任务：
  `/task 完成任务 #18G-1001`
- 添加备注：
  `/task 添加备注 #18G-1001 已完成联调`
- 查看进行中：
  `/task 查看进行中`

> 具体可用语法以 bot 当前实现为准。

---

## 6. 账号管理建议

- 至少保留 1 个 superadmin 账号
- 业务账号按岗位创建，不共用密码
- 定期禁用离职账号
- 管理员账号强制高强度密码与定期轮换

---

## 7. 安全与运维建议

- 不要把 `.env`、数据库、导出文件提交到 Git
- 生产部署加 HTTPS 与访问控制
- 定时备份 `task_system/data.sqlite3`（或数据文件）
- 升级前先打包与校验（建议保留 SHA256）

---

## 8. 快速故障定位

1. API 存活：`curl http://127.0.0.1:8090/health`
2. 端口监听：`ss -lntp | grep 8090`
3. Bot 日志：查看 bot 进程输出 / service 日志
4. 数据文件权限：确认运行用户可读写 `task_system/`

如仍异常，建议收集：报错截图 + 最近日志 + 复现步骤，再统一排查。
