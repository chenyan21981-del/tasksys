# SECURITY.md

## 1) 威胁模型（简版）

### 资产
- 任务数据（主任务、子任务、备注、流转记录）
- 账号与权限（admin/superadmin）
- Telegram Bot Token 与会话信息

### 主要攻击面
- Web 登录与 API（`/auth/*`, `/tasks*`, `/admin/*`）
- 服务监听地址与网络暴露
- 部署/安装脚本（服务文件、环境变量、初始账号）
- 日志与运维脚本（信息泄露风险）

### 主要风险
- 弱口令/明文凭据泄露
- 未加密公网传输导致凭据被窃听
- 误将服务绑定到公网接口
- 高权限自动化变更（未经人工审核）

---

## 2) 安全基线（当前实现）

- 默认监听：`127.0.0.1`（可通过 `TASK_BIND_HOST` 覆盖）
- 初始管理员密码：安装时随机强密码，一次性展示
- 存储方式：`PBKDF2` 哈希（不落盘明文）
- 安装脚本：不自动执行 `sudo` 安装系统服务，需人工审核后手动执行
- 访问信息脚本：不再输出 admin 密码

---

## 3) 部署红线（必须遵守）

1. **禁止**将 HTTP 明文服务直接暴露公网。
2. 对外访问必须走 **HTTPS 反向代理**（Nginx/Caddy/Traefik 均可）。
3. 未完成凭据轮换前，禁止交付生产使用。
4. 禁止在聊天、工单、日志中粘贴 Bot Token / 管理员密码。
5. 涉及 systemd、网关、鉴权的改动，必须先备份并可回滚。

---

## 4) 推荐生产部署

## 网络
- 应用仅监听内网/本机：`TASK_BIND_HOST=127.0.0.1`
- 防火墙仅放行反向代理端口（80/443）
- 8090 仅本机或内网可达

## 反向代理
- 强制 HTTPS（TLS 1.2+）
- 开启 HSTS（按需）
- 限制请求体大小与基础限流

## 鉴权
- 管理员首次登录立即改密
- 建议密码策略：>=14 位，大小写+数字+符号
- 高权限账号最小化，定期审计启用状态

## 密钥管理
- Token/Secret 仅存放在环境变量或受控配置文件
- 泄露即轮换
- 避免默认值作为生产值

## 监控与日志
- 记录登录失败、权限变更、关键任务操作
- 日志避免输出敏感字段（密码、token、cookie）
- 建立最小告警：服务离线、连续登录失败、异常 5xx 峰值

## 5) 运维检查清单（上线前/每周）

### 上线前（一次）
- [ ] `ss -ltnp | grep 8090` 确认非公网监听
- [ ] 反向代理 HTTPS 正常，证书有效
- [ ] admin 密码已修改为强密码
- [ ] Bot Token 已替换默认/旧值
- [ ] 备份与回滚流程已演练

### 每周巡检
- [ ] 检查异常登录/错误日志
- [ ] 检查账号启用状态与权限漂移
- [ ] 检查服务状态与端口暴露
- [ ] 检查系统补丁与 Node 版本

---

## 6) 应急处置（最小流程）

1. 发现泄露：立即轮换密码与 Token
2. 临时收敛：将监听收敛到 `127.0.0.1`，下线公网入口
3. 保全证据：导出关键日志（时间线）
4. 恢复服务：按“备份→修改→验证”流程恢复
5. 复盘：补齐规则与自动检查
